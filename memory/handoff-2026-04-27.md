## Key Decisions

- **Two products, not one**: moth = Jira MCP server. Workflow MCP (CSV/JSON/search/file-info) is a separate product in a separate folder. Why: different audiences, different value props, would have been scope creep together.
- **Jira first, Confluence/Bitbucket post-v1**: Jira has the clearest market gap and broadest addressable market. Confluence write operations are actively dangerous in the Atlassian server. Bitbucket is a narrower audience. Rejected: starting with Bitbucket (smaller market) or Confluence (risky write surface).
- **Build from scratch, not a fork**: `atlassian/atlassian-mcp-server` is config pointing to a closed hosted server ŌĆö no source to fork. Confirmed by direct repo inspection.
- **Remote HTTP/SSE, not local stdio**: Hosted product, users paste a URL. This is the right model for a commercial product.
- **Single `PROJECT_ROOT` for the workflow MCP** (if built): Replaces the 5 `KADUKA_*_DIR` env vars from the source repo. Identifier system (api/web/scripts/support/mcp) dies entirely.
- **"Kaduka" name must be fully excised**: Robert does not own the Kaduka trademark. The kaduka_mcp source is reference only.
- **RFC 9728 resource metadata is the auth fix**: Atlassian's server breaks differently per client because it doesn't implement `/.well-known/oauth-protected-resource`. Implementing it correctly is the differentiator.
- **No retry on createJiraIssue**: The double-create bug (#132) is caused by a retry/double-invocation pattern. Single POST only.

## Open Questions

- **Token storage (`src/auth/tokens.ts`)**: The in-memory `Map` must be replaced before production. Options: Redis (fast, ephemeral by default, requires infra), Postgres (durable, already needed for billing), or encrypted cookies (no server-side storage, but size limits). Decision criteria: whether billing requires a DB anyway (likely yes), and ops budget. If Stripe is the billing choice, Postgres is the natural fit ŌĆö one DB for both.

- **Hosting platform**: Spec says Railway or Fly.io. Neither has been evaluated. Decision criteria: cold start behavior (SSE connections need fast startup), pricing at low volume, ease of Postgres add-on. Both support Node.js natively.

- **Atlassian OAuth app registration**: No OAuth app has been registered yet at developer.atlassian.com. This is required before the OAuth flow can be tested end-to-end. `ATLASSIAN_CLIENT_ID` and `ATLASSIAN_CLIENT_SECRET` are currently empty strings in config.

- **`getJiraDevPanel` (`moth-meg.10`)**: The development information API endpoint is not standard Jira REST API v3. It may require a different base URL or additional OAuth scopes. Needs investigation before implementation.
